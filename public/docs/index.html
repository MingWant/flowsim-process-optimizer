🎯 三種模式快速對比

📊 Both

顯示：Calendar、Working、Diagnostic 指標

適合：完整分析、學習階段、不確定時

優點：資訊最全面

📅 Calendar Time

顯示：日曆時間（含非工作時間）

適合：客戶報告、SLA 檢查

優點：客戶視角、對外溝通

⏱️ Working Time

顯示：完成項目的真實工作時間等待

適合：資源規劃、內部優化

優點：item-weighted 佇列效率

🚀 快速開始

第一次使用（5 分鐘）

  • ✓ 閱讀 快速參考（5 分鐘）
  • ✓ 開啟系統 → 左側邊欄 → "Wait Time Calculation"
  • ✓ 選擇 📊 Both 模式
  • ✓ 觀察統計面板的兩個等待時間指標

準備客戶報告

  • ✓ 切換到 📅 Calendar Time 模式
  • ✓ 查看/匯出 Calendar Wait 數據
  • ✓ 用於 SLA 報告和客戶溝通

內部優化決策

  • ✓ 切換到 ⏱️ Working Time 模式
  • ✓ 查看 Working Wait 數據
  • ✓ 根據佇列效率決定是否增加資源

💡 關鍵概念

週五 17:00 入隊 → 週一 9:00 處理 📅 Calendar Wait = 64 小時 （客戶：「我等了整個週末！」） ⏱️ Working Wait = 0 小時 （營運：「佇列沒有積壓」） 🩺 Diagnostic Working Wait 步驟級平均，用於找瓶頸；不是每個 item 的真實平均 差異 = 64 小時 = 週末時間

如何選擇模式？

你在做什麼？ 選擇模式
給客戶看 / SLA 報告 📅 Calendar Time
內部優化 / 資源規劃 ⏱️ Working Time
完整分析 / 不確定 📊 Both

📋 常見問題

Q: 切換模式會改變資料嗎？

A: ❌ 不會，只改變顯示。後台始終計算兩種等待時間，切換只是改變 UI 顯示哪些卡片。

Q: 哪個模式最準確？

A: ✅ 都準確，只是視角不同。Calendar Time = 客戶視角（完整等待），Working Time = 營運視角（佇列效率）。

Q: Queue Wait (Working) 和 Diagnostic Working Wait 差在哪？

A: Queue Wait (Working) 按完成 item 的真實經歷聚合；Diagnostic Working Wait 是步驟級平均，適合找瓶頸。

Q: 兩個指標差異很大，正常嗎？

A: ✅ 完全正常。差異大表示非工作時間影響大（如週末），差異小表示主要是佇列積壓。

Q: 模式選擇會儲存嗎？

A: ⚠️ 目前不會，重新整理頁面會重設為預設模式（Both）。
