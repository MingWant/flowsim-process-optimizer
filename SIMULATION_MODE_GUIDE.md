# FlowSim 模拟模式使用指南

## 📊 两种模拟模式对比

FlowSim 现在支持两种模拟模式，用于不同的分析场景：

### 🟢 **真实模式 (Realistic Mode)** - 默认

适用于：日常流程模拟、性能分析、平均场景预测

**特点**：
- ✅ 处理时间使用**正态分布**（钟形曲线）
- ✅ 队列取消使用**指数分布**（泊松过程）
- ✅ 批量到达有**微小时间偏移**（0.1ms间隔）

**行为**：
```
处理时间分布：大部分接近平均值，极端情况罕见
队列取消：渐进式增长，即使等很久也有人留下
批量到达：物品有物理先后顺序
```

**示例场景**：
- 预测日常吞吐量
- 分析平均周期时间
- 优化正常运营下的瓶颈

---

### 🟠 **最坏情况模式 (Worst-Case Mode)**

适用于：容量规划、压力测试、极端场景分析

**特点**：
- ⚠️ 处理时间使用**均匀分布**（可能出现极端值）
- ⚠️ 队列取消使用**线性模型**（快速达到100%）
- ⚠️ 批量到达**同一瞬间**（瞬时压力峰值）

**行为**：
```
处理时间分布：极快和极慢的情况同样常见
队列取消：短时间内大量离开（测试高压情况）
批量到达：所有物品同时到达（最大瞬时压力）
```

**示例场景**：
- 确定需要的最大容量
- 测试系统在极端压力下的表现
- 保守的容量规划（安全边际）

---

## 🎯 何时使用哪种模式？

### 使用真实模式当：
- ✓ 你想了解**平均**性能
- ✓ 你在做日常运营优化
- ✓ 你想识别**正常情况**下的瓶颈
- ✓ 你需要向管理层展示**现实**预期

### 使用最坏情况模式当：
- ✓ 你在做**容量规划**（需要留安全边际）
- ✓ 你想测试系统能承受的**最大压力**
- ✓ 你在准备应对**突发事件**（如促销高峰）
- ✓ 你需要保守的估算来说服采购更多资源

## 📈 具体差异示例

### 1️⃣ **处理时间变异 (variance = 0.3)**

**真实模式**（正态分布）：
```
基础时间: 1000ms
实际时间分布：
- 68% 在 700-1300ms 之间
- 95% 在 400-1600ms 之间
- 99.7% 在 200-1800ms 之间（硬限制20%-300%）
极端快/慢的情况：罕见
```

**最坏情况模式**（均匀分布）：
```
基础时间: 1000ms
实际时间分布：
- 所有值在 700-1300ms 之间均匀分布
- 极端值（700ms 或 1300ms）出现概率与平均值相同
- 可能产生接近0的极端快速情况
```

---

### 2️⃣ **队列取消 (cancellationProbability = 0.1)**

**真实模式**（指数分布）：
```
等待 5秒:  39.3% 离开
等待 10秒: 63.2% 离开
等待 20秒: 86.5% 离开
等待 50秒: 99.3% 离开
等待 100秒: 99.995% 离开（仍有0.005%留下）
```

**最坏情况模式**（线性）：
```
等待 5秒:  50% 离开
等待 10秒: 100% 离开（全部离开）
等待 15秒: 100% 离开
等待 50秒: 100% 离开
```

⚠️ **最坏情况模式**：10秒后队列**强制清空**，模拟极端不耐烦场景

---

### 3️⃣ **批量到达 (batchSize = 10)**

**真实模式**：
```
时间 10:00:00.000 - 物品1到达
时间 10:00:00.001 - 物品2到达
时间 10:00:00.002 - 物品3到达
...
时间 10:00:00.009 - 物品10到达

→ 单资源瓶颈的等待时间统计更准确
```

**最坏情况模式**：
```
时间 10:00:00.000 - 物品1到达
时间 10:00:00.000 - 物品2到达
时间 10:00:00.000 - 物品3到达
...
时间 10:00:00.000 - 物品10到达

→ 测试瞬时10个物品同时冲击的场景
```

## 🔧 如何切换模式

### 在UI中：
1. 打开左侧边栏
2. 在"Quick Controls"部分找到"Simulation Mode"
3. 点击"Realistic"或"Worst-Case"按钮

### 通过代码：
```typescript
setConfig(prevConfig => ({
  ...prevConfig,
  simulationMode: 'worst-case' // 或 'realistic'
}));
```

## 💡 实际应用建议

### 场景1：日常优化
```
问题：我们的订单处理流程哪里最慢？
建议：使用真实模式
原因：反映真实情况，找到实际瓶颈
```

### 场景2：招聘决策
```
问题：明年订单量增长50%，需要多少员工？
建议：先用真实模式，再用最坏情况验证
原因：真实模式给出期望值，最坏情况确保有安全边际
```

### 场景3：促销准备
```
问题：双11活动期间能否应对流量？
建议：使用最坏情况模式
原因：模拟批量订单同时涌入，取消率高，处理时间极端
```

### 场景4：向管理层汇报
```
问题：需要展示新流程的改进效果
建议：使用真实模式
原因：真实数据更可信，避免过度保守的印象
```

## 🔬 技术细节

### 正态分布 vs 均匀分布

**正态分布**（Box-Muller变换）：
```typescript
const u1 = Math.random();
const u2 = Math.random();
const normalRandom = Math.sqrt(-2 * Math.log(u1)) * Math.cos(2 * Math.PI * u2);
const duration = base + base * variance * normalRandom;
```

**均匀分布**（原算法）：
```typescript
const uniformRandom = Math.random() * 2 - 1; // -1 到 +1
const duration = base * (1 + uniformRandom * variance);
```

### 指数分布 vs 线性模型

**指数分布**（泊松过程）：
```typescript
cancelChance = 1 - Math.exp(-probability * (timeInSeconds));
```

**线性模型**（原算法）：
```typescript
cancelChance = Math.min(1, probability * timeInSeconds);
```

## ⚠️ 注意事项

1. **最坏情况不是"更准确"**，它是**更保守**
2. **不要混合使用**：同一分析中始终使用同一模式
3. **文档记录**：导出时会保存使用的模式
4. **对比验证**：建议两种模式都跑一遍，观察差异

## 📊 预期差异

在相同配置下，两种模式的典型差异：

| 指标 | 真实模式 | 最坏情况模式 | 差异 |
|-----|---------|-------------|-----|
| 平均周期时间 | 基准 | +10-30% | 更长 |
| 最大队列长度 | 基准 | +50-200% | 更长 |
| 取消数量 | 基准 | +100-500% | 更多 |
| 吞吐量 | 基准 | -10-30% | 更低 |
| 资源利用率峰值 | 基准 | +20-50% | 更高 |

## 📝 导出和分享

导出的JSON文件会包含模式信息：

```json
{
  "app": "FlowSim",
  "version": 3,
  "config": {
    "simulationMode": "worst-case",
    "steps": [...]
  }
}
```

这样其他人导入时会保持你使用的模式。

## 🎓 总结

- 🟢 **真实模式**：用于理解和优化日常运营
- 🟠 **最坏情况**：用于容量规划和压力测试
- 🔄 **两者结合**：全面了解系统在各种情况下的表现

选择正确的模式是准确分析的第一步！
