# 等待時間模式 - 快速參考

## 🎯 三選一

### 📊 Both (預設)
```
顯示：兩個指標
適合：完整分析、學習階段
```

### 📅 Calendar Time  
```
顯示：日曆時間（含週末/夜間）
適合：客戶報告、SLA 檢查
```

### ⏱️ Working Time
```
顯示：工作時間（純佇列效率）
適合：資源規劃、內部優化
```

---

## 📍 設定位置
左側邊欄 → **Wait Time Calculation**（在 Simulation Mode 下方）

---

## 💡 快速決策

| 我要做... | 選哪個 |
|---------|--------|
| 🔍 全面分析 | 📊 Both |
| 📄 客戶報告 | 📅 Calendar |
| 🔧 內部優化 | ⏱️ Working |
| ❓ 不確定 | 📊 Both |

---

## 🔍 關鍵差異

**範例：週五晚上入隊 → 週一早上處理**

```
📅 Calendar Time = 64 小時
   （客戶：「我等了整個週末！」）

⏱️ Working Time = 0 小時
   （營運：「佇列沒有積壓」）

差異 = 64 小時 = 週末
```

---

## ✅ 使用技巧

### 客戶 SLA 報告
1. 切換到 📅 **Calendar**
2. 查看/匯出資料
3. 完成後切回 📊 **Both**

### 決定加不加人
1. 切換到 ⏱️ **Working**
2. 如果 > 目標 → 需要加人
3. 如果 < 目標 → 不需要

### 對比工作時間配置
1. 切換到 ⏱️ **Working**
2. 測試配置 A
3. 測試配置 B
4. 對比（公平，因為都看佇列效率）

---

## 🎓 記住這個

> **Calendar Time - Working Time = 非工作時間**

差距大 → 主要是週末/夜間導致 → 考慮延長工作時間  
差距小 → 主要是佇列積壓 → 需要增加資源

---

## 📋 常見問題

**Q: 切換會改變資料嗎？**  
A: ❌ 不會，只改變顯示

**Q: 哪個模式最準？**  
A: ✅ 都準，只是視角不同

**Q: 會儲存我的選擇嗎？**  
A: ⚠️ 目前不會，重新整理會重設

---

## 🚀 一句話總結

**給客戶看 → Calendar | 內部優化 → Working | 不確定 → Both**
